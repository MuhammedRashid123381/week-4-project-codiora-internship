Portfolio Website - Week 5

This is a personal portfolio website built to showcase my projects, skills, and experience. It was developed over five weeks, with this final phase focused on production optimization, accessibility, SEO, and public deployment.

Overview

The portfolio highlights my work through a clean, modern interface with interactive components. This week's focus was on preparing the project for real-world use rather than adding new features, including deployment, performance tuning, SEO, accessibility, and code cleanup.

Features

The website is fully responsive across desktop, tablet, and mobile devices. It includes a dynamic project showcase, smooth navigation, optimized and lazy-loaded images, SEO-friendly meta tags with Open Graph support, and accessibility features such as keyboard navigation, ARIA labels, and alt text. The codebase is organized into clean, reusable components.

Technologies used

React or Next.js for the frontend, Tailwind CSS for styling, Vercel or Netlify for deployment, Git and GitHub for version control, and Google Lighthouse for performance and accessibility auditing. Update this list to reflect your actual stack.

Installation and setup

Clone the repository with git clone followed by your repository URL. Move into the project folder with cd and the folder name. Install dependencies with npm install. Start the local server with npm run dev. The site will then run at localhost on port 3000 by default.

Deployment

The project is deployed using Vercel or Netlify. To deploy your own copy, push the project to GitHub, connect the repository to your chosen hosting platform, configure any build settings needed, then deploy and share the resulting public link.
